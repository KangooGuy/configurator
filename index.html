Port 136 — GNSS DOP Configuration

Configure GNSS dilution-of-precision (DOP) thresholds and cool-off timer for Tag S. Payload is 4 bytes.

Configuration Fields

DOP values use 0.1 granularity — store 10× the actual DOP (e.g. 2.5 → enter 25).
FieldDecimal (raw ×10)Hex
Cool-off timer (s, 0–59) [uint8]
PDOP × 10 (0=0.0 … 255=25.5) [uint8]
HDOP × 10 (0=0.0 … 255=25.5) [uint8]
VDOP × 10 (0=0.0 … 255=25.5) [uint8]

Bytestring

Live Byte Preview

DOP thresholds: Lower values enforce stricter GNSS fix quality.
Default 50 = DOP 5.0 — falls back to cool-off timer if no threshold is met.
Cool-off timer: Seconds to wait between successive GNSS fix attempts.

Bit mapping: Bit 7–6: reserved · Bit 5: GNSS · Bit 4: WiFi · Bit 3: ACC · Bits 2–0: Datarate
Datarate: 000=DR5(SF7) · 001=DR4(SF8) · 010=DR3(SF9) · 011=DR2(SF10) · 100=DR1(SF11) · 101=DR0(SF12) · 110=DR1–3 array [DEFAULT]
Value 111 (ADR) is deprecated in current firmware — do not use.
